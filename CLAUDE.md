## Project Overview

This is the **Mass. Numbers** blog (www.massnumbers.us) — a data-driven blog about Massachusetts politics and policy. It is built with [Quarto](https://quarto.org/) and hosted on GitHub Pages.

## Architecture

- **`_quarto.yml`** — Quarto site configuration (title, navbar, output directory, theme, Google Analytics, social metadata)
- **`posts/`** — Blog post source directories, each named `YYYY-MM-DD-slug/` containing an `index.qmd` file and any post-specific CSV data files
- **`docs/`** — Compiled HTML output (the published site). Generated by `quarto render`; do not edit directly
- **`_data/`** — Shared data files (CSV, XLSX) organized by year subdirectories (`2020/`, `2023/`, `2024/`)
- **`_gis/`** — GIS shapefiles and GeoJSON for Massachusetts districts, towns, precincts, and neighborhoods
- **`images/`** — Site images and post preview images
- **`massnumbers.scss`** — Custom Quarto SCSS theme (Amiri font, blue header)
- **`_freeze/`** — Quarto computation cache (committed to git; avoids re-running R code on every render)
- **`CNAME`** — Custom domain: www.massnumbers.us

## Building the Site

Requires [Quarto CLI](https://quarto.org/docs/get-started/) and R with relevant packages.

```bash
# Render the entire site
quarto render

# Preview with live reload
quarto preview

# Render a single post
quarto render posts/YYYY-MM-DD-slug/index.qmd
```

Output goes to the `docs/` directory which is served by GitHub Pages.

## Blog Post Structure

Each post `index.qmd` follows this pattern:
1. YAML front matter with title, description, author, date, and `image` for social cards/listings
2. Setup chunk loading R libraries (global `echo: false` is set in `_quarto.yml`)
3. Data loading from local CSVs or `_data/` directory
4. Prose interspersed with R code chunks producing tables and maps
5. Wide-layout chunks use `#| column: body-outset`, `#| column: page`, or `#| column: screen`

Social metadata (Open Graph, Twitter cards) is generated automatically by Quarto from the post YAML and site-level `_quarto.yml` config.

## Commonly Used R Packages

- **tidyverse** — data wrangling (used in every post)
- **sf** / **tmap** / **tmaptools** — spatial data and interactive maps (Leaflet-based)
- **gt** / **gtExtras** — formatted tables
- **plotly** — interactive charts
- **formattable** — formatted numeric display
- **rstanarm** — Bayesian regression (used in some analytical posts)

## Content Domain

Posts analyze Massachusetts election data: precinct-level results, PVI (Partisan Voter Index) ratings, census demographics, state legislative district maps, campaign finance (OCPF) data, and special election previews. Data files use MA-specific terminology (VTDs, precincts, state rep/senate districts, congressional districts).
